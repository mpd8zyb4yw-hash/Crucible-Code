How this works

Methodology

These numbers come straight from Crucible's boot smoke suite — the same automated checks that run when the engine starts. There is no cherry-picking and no hand-tuned demo set: this page shows the most recent run exactly as the suite reported it.

Each category is graded pass / fail by the suite. Hard failures are blocking regressions; soft failures are degradations that don't block boot. The overall state is the suite's own pass flag and process exit code. A value of -1 means that count was not reported by this run.

Data is published to /api/benchmarks/publish (authenticated) and read back here from /api/benchmarks/public. The page is fully static; it just fetches the latest snapshot. Refresh to re-pull.
