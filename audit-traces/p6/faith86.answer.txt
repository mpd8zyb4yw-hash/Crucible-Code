## Zod Schema for IPv4 Address Validation

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "IPv4 Address",
  "type": "object",
  "properties": {
    "ip": {
      "type": "string",
      "pattern": "^(25[0-5]|2[0-4][0-9]|[01]?[0-9]{2}){3}$"
    }
  },
  "required": ["ip"]
}
```

## Explanation

- **$schema**: Specifies the JSON Schema version.
- **title**: The title of the schema.
- **type**: The type of the schema, which is an object.
- **properties**: A key-value pair where each key is a property name and the value is the schema for that property.
  - **ip**: A string representing the IPv4 address.
    - **type**: String.
    - **pattern**: A regular expression that matches a valid IPv4 address. It ensures the address has exactly four octets, each ranging from 0 to 255.
- **required**: An array of property names that are required for the schema to be valid.

This schema validates an IPv4 address by checking that it consists of exactly four octets, each ranging from 0 to 255.
